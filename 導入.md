# 1. このコンペでやること

## 目的

ポケモンカードの対戦データを使って、

「player1 が勝つ確率」

を予測する。

---

## イメージ

| player1 | player2 |
|---|---|
| リザードンデッキ | ミライドンデッキ |

↓

AIに

「player1が勝つ確率は 0.73」

みたいに予測させる。

---

# 2. Kaggleでやっていることの本質

Kaggleでは基本的に、

```python
データを見る
↓
特徴を見つける
↓
機械学習モデルに学習させる
↓
予測する
```

をやっている。

---

# 3. このコンペのデータ構造

データは複数ファイルに分かれている。

---

# 4. train と test

## train

学習用データ。

答え(target)が入っている。

```python
この試合はplayer1が勝った
```

みたいな正解付き。

---

## test

予測対象。

答え(target)が無い。

AIが予測する。

---

# 5. targetとは？

## target

予測したいもの。

| target | 意味 |
|---|---|
| 1 | player1 勝利 |
| 0 | player2 勝利 |
| -1 | 不戦勝(bye) |

---

# 6. 評価指標 ROC-AUC

## ROC-AUCとは？

予測の上手さを表す数字。

| 値 | 意味 |
|---|---|
| 0.5 | ランダム |
| 0.7 | 結構良い |
| 0.8以上 | 強い |
| 0.9以上 | 超強い |
