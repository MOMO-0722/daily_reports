# 「データ理解」って何をするの？

Kaggle初心者だと、

```python
とりあえずモデル作ればいい？
```

ってなりやすい。

でも実際は、

```python
まずデータを理解する
```

が超重要。

---

# このコンペで与えられているデータ

大きく分けると5種類。

| ファイル | 役割 |
|---|---|
| train_matches | 試合結果 |
| test_matches | 予測対象試合 |
| deck_train | デッキの中身 |
| deck_test | test側デッキ中身 |
| card_list | カード詳細情報 |

---

# 1. train_matches

## これがメイン

```python
1行 = 1試合
```

---

## 例えば

| player1 | player2 | target |
|---|---|---|
| リザードン | ミライドン | 1 |

意味：

```python
player1が勝った
```

---

## 主な列

| 列名 | 意味 |
|---|---|
| player1_id | プレイヤー1 |
| player2_id | プレイヤー2 |
| player1_deck_id | デッキID |
| player2_deck_id | デッキID |
| player1_archetype | デッキタイプ |
| player2_archetype | デッキタイプ |
| target | 勝敗 |

---

# 2. test_matches

## 予測する相手

構造はtrainとほぼ同じ。

---

## でも違う

```python
targetが無い
```

---

## なぜ？

AIが予測するから。

---

# 3. deck_train

ここ超重要。

---

# deck_train の意味

```python
1行 = デッキに入ってるカード1種類
```

---

## 例

| deck_id | card_name | count |
|---|---|---|
| deck001 | ナンジャモ | 4 |
| deck001 | ボスの指令 | 2 |

---

## つまり

```python
このデッキには
ナンジャモ4枚入ってる
```

という意味。

---

# deck_train を理解すると何が分かる？

## デッキ構成

例えば：

```python
このデッキは炎エネルギー多い
```

とか。

---

## 特徴

```python
サポート多め
```

とか。

---

## 強さ傾向

```python
このカード入ってると勝率高い
```

とか。

---

# 4. card_list

## カード辞典

カードの詳細情報。

---

## 例えば

| card_name | hp | types |
|---|---|---|
| リザードン | 330 | Fire |

# 分かること

- HP
- タイプ
- 技
- 弱点
- 進化元

など。

---

# 5. card_list_improved

強化版カード辞典。

---

## 追加情報

- rarity
- ability
- resistance
- flavor_text

など。
